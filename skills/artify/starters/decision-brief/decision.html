A reusable shell for choosing between real options, not just writing vibes in a memo.

Decision flow
Decision

Should we ship a hybrid document starter system as a first-class lane in html-artifact?

This starter is for product bets, vendor picks, staffing calls, tooling changes, pricing moves, or any other choice that benefits from explicit criteria and a recommendation.

Recommended

Yes, ship the hybrid lane now.

It unifies long-read and presentation work into one artifact, which lowers authoring friction and makes the skill more distinct.

Decision deadline

End of sprint, so starter naming and file structure stay clean before more examples accrete like mold.

Presenter note: say the recommendation in the first thirty seconds so the room knows what is being asked of them.
Options

Compare the paths honestly.

Option A

Keep the existing library as-is. Lowest effort, weakest differentiation.

Option B

Add a few hybrid examples only. Better signal, but still fuzzy taxonomy.

Option C

Define a hybrid starter family with shared read, focus, and slides behavior.

What changed compared to the old framing?

We stopped thinking in dead file types and started thinking in reusable document moves plus presentation-ready shells.

Criteria

Score against criteria that matter.

Criterion Weight Option A Option B Option C
Reuse speed 30% 2 3 5
Presentation quality 25% 2 4 5
Library coherence 25% 2 3 5
Build effort 20% 5 4 3
Scoring note

Use a 1-5 scale, document assumptions, and add a sensitivity note if one weight change could flip the answer.

Recommendation

Close with a decision, not a shrug.

Recommended path

Adopt Option C and launch with a broad starter set so users immediately understand the new lane.

Risk and mitigation

The main risk is inconsistency across files. Mitigate by giving every starter the same mode toggles and section logic.
