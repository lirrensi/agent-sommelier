Onboarding Path

For new hires, client onboarding, partner enablement, or any role-based setup path.

1/4
Path flow
Start here

Orient the person fast.

This starter helps someone understand where they are, what they must do first, and what changes by role.

Day one goal

Get access, know the map, and complete the first meaningful task.

Success signal

The person can act independently without hunting for six missing docs.

Presenter note: start with outcomes, not tool names. Nobody bonds emotionally with a permissions matrix.
Role paths

Filter the path by role.

Builders

Set up tooling, clone repos, run the first local task, and learn where implementation notes live.

Managers

Learn approval flows, planning rituals, reporting cadence, and how decisions get documented.

Partners

Review access boundaries, shared deliverables, communication norms, and escalation paths.

Checkpoints

Use milestones instead of walls of instructions.

Checkpoint 1

Access confirmed and workspace opened.

Checkpoint 2

First role-specific task completed with help nearby.

Checkpoint 3

Knows where to ask, where to read, and where to log progress.

Resources

Repository guide, operating principles, common commands, decision records.

People

Owner, onboarding buddy, escalation contact, and subject experts.
